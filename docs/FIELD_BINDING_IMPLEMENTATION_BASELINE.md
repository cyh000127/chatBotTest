# Field Binding Implementation Baseline

## 1. 목적

이 문서는 farmer-facing runtime에서 농지 바인딩과 `/myfields` 조회를 구현할 때 따라야 하는 최소 기준을 정리한다.

핵심 목표는 다음과 같다.

- 위치 공유 기반의 실제 농지 후보 조회를 제공한다.
- 고유 번호 기반 fallback을 제공한다.
- 자동 확정이 어려운 경우 운영 검토로 넘긴다.
- `/myfields`가 안내 문구가 아니라 실제 참여자별 농지 목록을 보여주도록 만든다.

## 2. 범위

현재 단계에서 구현하는 범위는 아래와 같다.

- published 상태의 농지 레지스트리 버전 관리
- 농지 경계 polygon 저장
- 위치 좌표 기준 후보 조회
- 고유 번호 입력 fallback
- 참여자와 농지의 바인딩 저장
- 중복, 충돌, 다중 후보 상황의 예외 저장
- `/myfields` 참여자 범위 조회

현재 단계에서 구현하지 않는 범위는 아래와 같다.

- 외부 지도 서버 연동
- 운영 콘솔에서 polygon 편집기 제공
- 대규모 배치 import 파이프라인
- 시즌, 이벤트, 증빙과의 완전한 연결

## 3. 기본 원칙

- `/myfields`는 승인된 참여자만 접근할 수 있다.
- 후보 조회는 published 레지스트리 버전만 사용한다.
- 위치 좌표가 polygon 내부에 들어간 경우를 우선 후보로 본다.
- 내부에 들어가는 후보가 여러 개면 자동 확정하지 않는다.
- 내부 후보가 없더라도 가까운 농지를 임의 확정하지 않는다.
- 자동 확정이 불가능하면 고유 번호 입력 또는 운영 검토로 유도한다.
- 고유 번호로 찾은 농지도 중복 바인딩 또는 충돌이 있으면 자동 확정하지 않는다.

## 4. 대화 흐름

### 4.1 `/myfields`

`/myfields` 진입 시 런타임은 아래 순서로 동작한다.

1. 현재 참여자의 활성 바인딩 농지 목록을 조회한다.
2. 바인딩된 농지가 있으면 목록과 최근 상태를 보여준다.
3. 바인딩된 농지가 없으면 비어 있는 상태와 등록 버튼을 보여준다.
4. 사용자는 위치 공유 또는 고유 번호 입력으로 등록을 시작할 수 있다.

### 4.2 위치 공유 등록

1. 런타임이 위치 공유를 요청한다.
2. 사용자가 위치를 보내면 published polygon에서 후보를 조회한다.
3. 후보가 1개면 확인 버튼을 통해 최종 확정한다.
4. 후보가 여러 개면 후보 선택 버튼을 먼저 보여준다.
5. 후보가 없으면 고유 번호 입력 fallback 또는 운영 검토로 유도한다.

### 4.3 고유 번호 fallback

1. 사용자가 고유 번호 입력을 선택한다.
2. 런타임은 고유 번호를 정규화한 뒤 published 레지스트리에서 찾는다.
3. 정확히 1개면 최종 확정한다.
4. 없거나 충돌하면 예외를 저장하고 운영 검토로 보낸다.

## 5. 저장 구조 기준

현재 단계의 SQLite 저장 구조는 아래 책임을 가져야 한다.

- `field_registry_versions`
  - 레지스트리 버전 단위 publish 상태 관리
- `field_registry_fields`
  - 농지 고유 번호와 표시 이름 저장
- `field_registry_boundaries`
  - polygon 좌표 저장
- `participant_field_bindings`
  - 참여자와 농지의 활성 바인딩 저장
- `field_binding_exceptions`
  - 다중 후보, 미일치, 중복 충돌 같은 자동 확정 실패 기록

예외는 바인딩을 대신하는 저장이 아니다.

예외는 운영 검토가 필요한 미해결 상태를 보존하는 용도다.

## 6. 자동 확정 규칙

자동 확정은 아래 조건을 모두 만족할 때만 허용한다.

- 승인된 참여자일 것
- published 버전에서 찾은 후보일 것
- 후보가 정확히 1개일 것
- 이미 다른 참여자에게 활성 바인딩된 농지가 아닐 것
- 같은 참여자에게 중복 바인딩이 아닐 것

하나라도 만족하지 못하면 예외 또는 fallback으로 내려간다.

## 7. 운영 검토 기준

아래 상황은 예외 저장 후 운영 검토 대상으로 분류한다.

- polygon 내부 후보가 여러 개
- 위치 공유로 후보를 찾지 못함
- 고유 번호가 없음
- 고유 번호가 여러 건으로 충돌
- 다른 참여자에게 이미 활성 바인딩된 농지

운영 검토는 기존 대화창과 별개 실시간 상담방을 만드는 기능이 아니다.

필요하면 운영자 후속 답변 또는 관리자 처리로 이어진다.

## 8. 완료 기준

- `/myfields`가 실제 참여자별 바인딩 목록을 조회한다.
- 위치 공유 기반 후보 조회가 실제 polygon 계산으로 동작한다.
- 고유 번호 fallback이 실제 레지스트리 조회로 동작한다.
- 자동 확정 불가 상황이 예외 저장과 운영 검토로 이어진다.
- 서버 재시작 후에도 바인딩과 예외 상태가 유지된다.
